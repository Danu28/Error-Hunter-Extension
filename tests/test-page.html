🧪 Error Hunter — Test Page

Use this page to manually verify Error Hunter captures all error types.

Test Instructions

  1. Open Chrome DevTools (F12) → Console tab
  2. Click the Error Hunter icon in the toolbar
  3. Click "Start Monitoring"
  4. Click each button below in order
  5. Open the Error Hunter popup and verify each error appears
  6. Try the Console and Network filter tabs
  7. Click individual errors to expand and see details
  8. Click "Clear" to verify it empties the list

1. Console Errors

These buttons trigger console.error() calls with various argument types.

console Each should appear as a JS Error in the popup.

2. Uncaught Exceptions

These trigger window.onerror via thrown exceptions.

exception Should appear as a JS Error with line/column info.

3. Unhandled Promise Rejections

These trigger unhandledrejection events.

rejection Should appear as a JS Error in the popup.

4. Failed Network Requests (Fetch)

These trigger intercepted fetch() calls to non-existent endpoints.

network Should appear as HTTP Error with status and method.

5. Failed Network Requests (XHR)

These trigger intercepted XMLHttpRequest calls.

network Should appear as HTTP Error with status and method.
